# Pilot 0.0.1 Experiment

## Goal

Train the first from-scratch Pilot model that can export to GGUF and load in
LM Studio, llama.cpp, or Ollama.

## Result

Status: failed as a useful chat model.

The model exports and loads successfully as a Llama-compatible GGUF, but it does
not produce stable Spanish or English chat responses. Simple prompts such as
`hola` and `como estas` can produce broken, unstable text.

## What Worked

- From-scratch Llama-style checkpoint trained locally.
- GGUF Q8_0 export works.
- LM Studio detects and loads the model.
- Chat template and EOS metadata are present in the GGUF.

## What Failed

- The model did not learn coherent chat behavior.
- Validation loss remained too high for reliable generation.
- The training run saw too few total tokens for a randomly initialized LLM.
- The data mix had too much generic web text and too little high-quality
  supervised chat data.
- There was no hard quality gate before importing into LM Studio.

## Diagnosis

This is a training-quality problem, not a C++ or GGUF problem. The model is
compatible with the runtime, but the weights are not good enough yet.

For `0.0.2`, the training pipeline should include:

- A larger and cleaner supervised chat set.
- A fixed eval suite with prompts like `hola`, `como estas`, translation,
  summary, and basic coding questions.
- Export only after local generation passes the eval prompts.
- More training tokens or a smaller first model that can overfit the chat format
  before scaling back up.
